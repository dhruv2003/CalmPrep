# CalmPrep

Your exam-season mental wellness companion.

## Overview
CalmPrep helps students monitor and improve their mental well-being during high-stakes board exams and competitive entrance tests (NEET, JEE, CUET, CAT, GATE, UPSC). It provides an empathetic, AI-powered space for daily mood logging and open-ended journaling.

**New Features:**
1. **Exam Panic Mode**: A keyboard-accessible, 3-step reset flow for overwhelm before, during, or after exams.
2. **Study Recovery Plan**: Practical, low-pressure academic recovery actions for the next 30 minutes, tonight, and tomorrow morning.
3. **Parent-Safe Summary**: A safe, non-diagnostic guardian overview that never exposes raw journal text.
4. **Burnout Radar**: Deterministic stress, sleep, study-hours, mood, and risk trends across recent check-ins.
5. **Micro-Journal Prompts**: Multilingual prompt chips that help students start writing without auto-submitting.

CalmPrep intentionally does **not** include AI voice, speech-to-text, text-to-speech, microphone permissions, or Gemini voice models. This keeps the product focused on exam stress support and reduces browser permission risk.

## Setup Instructions

1. **Install dependencies:**
   ```bash
   npm install
   ```

2. **Environment Variables:**
   Copy `.env.example` to `.env.local` and populate your Firebase and Gemini API keys.
   *Note: `GEMINI_API_KEY` is strictly server-side and should never be exposed to the frontend with `NEXT_PUBLIC_`.*
   ```bash
   cp .env.example .env.local
   ```

3. **Run the Development Server:**
   ```bash
   npm run dev
   ```

4. **Run Tests:**
   ```bash
   npm run test      # Watch mode
   npm run test:run  # Single run
   ```

## Firebase Notes
CalmPrep uses Firebase Auth (Google Provider) and Firestore. If the configuration is missing or invalid, the app gracefully falls back to **Demo Mode**, utilizing mock user profiles and mock wellness responses, allowing evaluators to test the UI without full configuration.

## AI Safety Notes
We take safety seriously. 
- The AI is instructed via prompt not to diagnose or claim to treat illnesses.
- Strict detection is active for signs of self-harm, suicidal ideation, or abuse, which instantly flags the check-in as `urgent` and `crisisSupportRequired: true`.
- An urgent risk will show an un-missable Safety Support card with emergency contacts and an option to notify a trusted guardian (if consent was given during onboarding).
- **Parent-Safe Summary**: Guardian alerts use a specialized, non-diagnostic summary generated by the AI, ensuring raw journal text and private details are never exposed to parents.
- Guardian alerts are consent-based, validated, and saved as `prepared` records unless email delivery is later connected through Firebase Functions or Cloud Run.
- Only one Gemini call is made per submit (Efficiency).

## PWA Configuration
This app is Progressive Web App (PWA) ready using `next-pwa`. It includes a manifest and responsive design out-of-the-box. It can be installed on mobile devices.

## Evaluation Alignment

This section details how CalmPrep aligns with the PromptWars Hackathon judging parameters:

### 1. Code Quality (High Impact)
- **TypeScript**: Used rigorously across all components and API routes. Strict types for all 5 new features.
- **Module Boundaries**: Dedicated helpers separate burnout logic, journal prompts, guardian alert safety, Gemini prompt building, and response validation.
- **Validation**: Schema-driven validation using `Zod` before the AI is invoked and to validate the response shape before sending it to the client.
- **Graceful Degradation**: The app works flawlessly in "Demo Mode" if API keys or Firebase configs are missing, ensuring it doesn't crash during evaluation.
- **Error Handling**: Uses Next.js `error.tsx` and `not-found.tsx` for robust global error boundaries.

### 2. Problem Statement Alignment (High Impact)
- Fully addresses the PromptWars brief: built specifically for NEET/JEE/etc., uncovering hidden stress triggers, emotional patterns, personalized coping strategies, adaptive mindfulness, and acting as an empathetic digital companion.
- **Micro-Journal Prompts** encourage students to reflect when they are overwhelmed.
- **Exam Panic Mode** provides a 3-step immediate grounding technique when anxiety peaks.
- **Study Recovery Plan** offers practical, low-pressure academic recovery steps.
- **Burnout Radar** aggregates historical check-ins to catch burnout early.
- **Parent-Safe Summary** bridges the gap between student privacy and guardian awareness.

### 3. Security (Medium Impact)
- **Server-Side API**: The Gemini API is called via a Next.js server route (`/api/wellness-check`), keeping the `GEMINI_API_KEY` entirely hidden from the browser.
- **Input Sanitization**: Journal text is sanitized to strip malicious tags before being embedded in prompts.
- **Privacy Preservation**: Guardian alerts use `guardianSafeSummary`, exclude raw journal text, avoid full wellness-result disclosure, validate guardian email, and respect consent.
- **Rate Limiting**: In-memory rate limiting prevents API abuse.

### 4. Efficiency (Medium Impact)
- **No Wasted Calls**: Exactly one Gemini call is made per submit. Burnout Radar uses the last 3-5 local/Firestore check-ins and deterministic helper logic instead of extra AI calls.
- **Singleton Pattern**: The `GoogleGenerativeAI` instance is hoisted to a singleton to avoid recreating it on every request.
- **Fast UI**: Built with pure Tailwind CSS, no heavy chart libraries, and no page-load Gemini requests.
- **Caching**: Mock mode utilizes `localStorage` for fast, efficient fallback reporting.
- **Streaming UI**: Uses `loading.tsx` to handle data loading gracefully.

### 5. Testing (Low Impact)
- Configured with `vitest`. Contains 53 focused tests.
- Covers response validation, prompt safety rules, burnout helper logic, guardian alert privacy, journal prompts, prompt-chip insertion, i18n coverage, and edge cases.

### 6. Accessibility (Low Impact)
- Semantic HTML tags (`<main>`, `<nav>`, `<form>`).
- High contrast "Calm Neobrutalism" UI (solid borders, clear text hierarchy).
- Exam Panic Mode uses `aria-expanded` and an `aria-live="polite"` reset panel.
- Micro-Journal Prompts are real keyboard-accessible buttons and never auto-submit.
- Form inputs perfectly paired with `htmlFor` and `id` tags.
- Dynamic `html lang` attribute updates when toggling languages.
- Screen reader announcements via `aria-live` for form errors and panic reset updates.
- Multilingual labels and prompts are available in English, Hindi, and Marathi.

## Evaluator Checklist

- No voice, speech-to-text, text-to-speech, microphone, or Gemini voice features.
- One Gemini call per check-in submit; no Gemini calls on report page load.
- Gemini key stays server-side only.
- Burnout Radar is deterministic and local, using recent check-ins without chart libraries.
- Guardian alerts are consent-based, email-validated, parent-safe, and never include raw journal text.
- Exam Panic Mode is calm, immediate, keyboard accessible, and uses `aria-live="polite"`.
- Micro-Journal Prompts are multilingual, editable starter prompts and do not submit the form.
